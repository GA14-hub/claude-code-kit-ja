# セキュリティルール

## 秘密情報の取り扱い

- `.env`、`secrets/`、認証情報ファイルを読み取らない
- API キー、トークン、パスワードをコードやコミットに含めない
- 秘密情報を出力やログに含めない
- ハードコードされた認証情報を見つけた場合は環境変数への移行を提案する

## コード生成時の注意

- SQL クエリにはプレースホルダ/パラメータバインディングを使う
- ユーザー入力は必ずバリデーション・サニタイズする
- 外部入力をそのままコマンド実行やファイルパスに渡さない
- HTTPS を前提とし、HTTP での通信を避ける

## 依存関係

- 非推奨や脆弱性のある依存パッケージの追加を避ける
- `node_modules/`、`vendor/` 等をコミットしない
