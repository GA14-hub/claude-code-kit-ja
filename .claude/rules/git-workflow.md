# Git ワークフロールール

## ブランチ運用

- main/master への直接コミット・push は禁止
- 作業は必ず feature ブランチで行う
- ブランチ名は `feature/〇〇`、`fix/〇〇`、`hotfix/〇〇` の形式

## コミット

- コミットメッセージは日本語で、変更内容が分かるように書く
- 1コミット = 1つの論理的な変更単位にする
- `git diff` で差分を確認してからコミットする
- `--force`、`--hard`、`--no-verify` は使わない

## プルリクエスト

- PR にはタイトル・概要・変更理由・テスト結果を含める
- レビューなしで main にマージしない
- マージ前にテストが通ることを確認する
