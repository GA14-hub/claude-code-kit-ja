# コーディングスタイルルール

## 基本方針

- プロジェクトの既存コードのスタイルに合わせる
- 新しいパターンを持ち込む前に既存パターンを確認する
- フォーマッタ/リンター（Prettier、ESLint、Black 等）の設定に従う

## 命名規則

- 変数名・関数名はその役割が分かる名前にする
- 略語を避け、読んで意味が通る名前を使う
- プロジェクトの言語慣習に従う（JavaScript: camelCase、Python: snake_case 等）

## コード構造

- 1つの関数/メソッドは1つの責務に絞る
- 関数が長くなりすぎたら分割を検討する
- マジックナンバーは定数として定義する
- コメントは「なぜ」を書く。「何を」はコード自体で表現する

## エラーハンドリング

- エラーを握りつぶさない（空の catch ブロックを避ける）
- エラーメッセージは原因と対処が分かる内容にする
